1. ビジョン / 製品の目的

「Google Drive バックアップ」 は、ブラウザ上で動作する Web アプリケーションである。 ユーザーが Google Drive 上に持つ任意のフォルダを、同じ親階層に「Copy of 元フォルダ名」として丸ごと複製する。

これにより「重要なフォルダを誤って編集・削除する前にスナップショットを取りたい」「特定のフォルダ構成を雛形として再利用したい」といったユーザーの欲求を、 ローカル PC のストレージを使わずに Google Drive 内で完結させる。

一言で言えば：「Drive 内で右クリック『コピー』をフォルダに対しても出来るようにする」ツールである。 Google Drive は標準ではフォルダ単位の複製機能を提供していない。本ツールはそのギャップを埋める。

2. ターゲットユーザー

ペルソナ 状況 本ツールに期待すること
個人 Drive ユーザー 大事な資料フォルダを編集する前に保険でコピーを残したい 1クリック相当でフォルダごと複製したい
小規模事業者・フリーランス 案件ごとに雛形フォルダ（請求書／契約書テンプレ等）を複製して新案件用に流用したい 毎回手作業でファイルを 1 つずつコピーする手間を省きたい
学生・研究者 論文・レポートの版管理として、提出前のフォルダ状態を保存したい ローカル PC を圧迫せずに「世代」を残したい

いずれも Google アカウントを持ち、ブラウザ（Chrome / Edge / Safari / Firefox 等）で日常的に Drive を使う一般ユーザーを想定。 プログラミング知識・API の知識は不要であること。

3. スコープと主要機能

3.1 MVP（最初に作る最小版）

「とにかく最小：1フォルダを一回コピーできる」を最優先する。

  • Google アカウントでログイン（OAuth2）
  • 複製したいフォルダの 共有 URL を貼り付けて指定
  • フォルダ情報（名前・所在・含まれるアイテム数）の事前プレビュー
  • 「複製を開始」ボタンで、「Copy of 元フォルダ名」を同じ親フォルダ内に作成し、再帰的に全アイテムを複製
  • 進捗表示（処理中のアイテム名・全体の進捗率）
  • 完了後にレポート表示（成功件数・スキップ件数・エラー詳細）

3.2 将来拡張（MVP には含めない）

  • 双方向同期（元フォルダの変更を複製先に追従させる）
  • 複数フォルダの一括複製・キュー処理
  • 世代管理（自動で日付付き複製を定期作成）
  • 共有設定の追従コピー
  • 共有ドライブ（チームドライブ）への対応

※ Picker API による GUI 選択方式は最小スコープ化と引き換えに URL 貼り付け方式と両立しないため、MVP・将来とも採用しない。フォルダ指定は共有 URL のペーストで統一する。

4. 成功定義（KPI）

観点 指標 目標
体験 初回ログインから複製開始までの操作数 5 クリック以内（ログイン含む）
体験 100 ファイル規模のフォルダの複製完了までの体感時間 3 分以内（API レート制限の範囲内で）
信頼性 複製成功率（個別ファイル単位） 95% 以上（残り 5% 以下はスキップでレポート）
整合性 複製後のフォルダ階層が元と一致する率 100%（成功したアイテムについて）
安全性 元フォルダ・元ファイルへの破壊的操作 0 件（読み取りと新規作成のみ）

5. ビジネス要件

  • BR-1: 個人ユーザーが無料で利用できるオープンな Web アプリとして提供する（課金機能は MVP に含めない）。
  • BR-2: ユーザーの Drive データはアプリのサーバには保存しない。OAuth トークンと API 経由のメタ操作のみで完結させる（プライバシー保護とサーバコスト抑制の両立）。
  • BR-3: Google の OAuth 利用ポリシーおよび Drive API の利用規約に準拠する。スコープは最小限（必要なファイル・フォルダの読み取りと新規作成）に限定する。
  • BR-4: 初回はクローズドベータ的に動作確認し、API クォータが許容範囲に収まることを確認してから一般公開する。

6. 機能要件

ID カテゴリ 要件 優先度
FR-01 認証 Google アカウントで OAuth2 認証を行い、Drive API の最小スコープのアクセストークンを取得できる High
FR-02 認証 ログイン状態が継続している場合、再認証なしで操作画面に進める Medium
FR-03 認証 ログアウト操作によりトークンを破棄し、未認証状態に戻せる Medium
FR-04 フォルダ指定 Drive フォルダの共有 URL を入力欄に貼り付けて指定できる。貼り付け直後に自動的にフォルダ情報の取得を開始する（手動「取得」ボタンは設けない） High
FR-05 フォルダ指定 貼り付けられた URL からフォルダ ID を抽出し、Drive API でメタデータ（名前・親フォルダ・直下のアイテム数）を取得して表示する。再帰的な総件数は表示しない（取得コスト削減のため） High
FR-06 フォルダ指定 無効な URL・存在しない ID・権限がないフォルダを指定した場合、明確なエラーメッセージを表示する High
FR-07 複製実行 「複製を開始」操作で、元フォルダと同じ親フォルダ内に「Copy of 元フォルダ名」という名前の新規フォルダを作成する。同名フォルダがすでに存在する場合は Copy of XXX (2)、(3) … のように連番を自動付与して衝突を回避する。元フォルダがマイドライブ直下にある場合はマイドライブ直下に複製先を作成する High
FR-07b 複製実行 非オーナーの編集可能フォルダを複製する場合、元フォルダと同じ親階層にユーザーが書き込み権限を持つときに限り複製先を作成する。書き込み権限がない場合は事前に検出して操作を中止し、エラーメッセージで通知する（自動的にマイドライブにフォールバックはしない） High
FR-08 複製実行 元フォルダ内のサブフォルダ・ファイルを再帰的に複製し、階層構造を保持する High
FR-09 複製実行 Google 独自ファイル（Docs / Sheets / Slides / Forms 等）は files.copy でそのまま複製する（PDF 等への変換はしない） High
FR-10 進捗表示 複製中、処理中のアイテム名・全体の進捗率（成功＋スキップ件数 ÷ 全件数）をリアルタイムに表示する High
FR-11 エラー処理 個別アイテムの複製に失敗した場合、エラー内容を記録してスキップし、全体の処理は続行する High
FR-12 完了レポート 全体完了時に、複製先フォルダへのリンク・成功件数・スキップ件数・スキップ理由の一覧を表示する High
FR-13 完了レポート レポート内容を CSV / プレーンテキストとしてダウンロードまたはコピーできる Low
FR-14 中断 複製処理中にユーザーが「中止」操作を行うと、進行中の API リクエストを AbortController で速やかに中断し処理を停止する。途中まで作成された複製先フォルダ（および複製済みのアイテム）は削除せずに残す Medium

7. 非機能要件

ID カテゴリ 要件
NFR-01 パフォーマンス 100 ファイル規模のフォルダで体感 3 分以内（Drive API のレート制限内で並列化を検討）
NFR-02 信頼性 API レートリミット（429 / 403 quotaExceeded）に対し指数バックオフでリトライする
NFR-03 セキュリティ OAuth スコープは drive.readonly + drive.file + openid email profile の組み合わせで確定。書き込みは本アプリが作成したフォルダ・ファイルに限定される（drive 全権限スコープは要求しない）。drive.readonly は URL 貼り付け方式の前提として必要
NFR-04 セキュリティ アクセストークンはブラウザのメモリ上でのみ保持し、サーバ・LocalStorage には保存しない（短期トークンの再取得方式）
NFR-05 プライバシー サーバ側でユーザーのファイル内容を保存・転送しない（クライアント完結 or サーバはトークン交換のみ）
NFR-06 可用性 ブラウザのみで動作し、追加インストール・拡張機能不要
NFR-07 互換性 Chrome / Edge / Firefox / Safari の最新版で動作（モバイルブラウザは MVP では非保証）
NFR-08 アクセシビリティ WCAG 2.1 AA を最低基準（キーボード操作・コントラスト・スクリーンリーダー対応）
NFR-09 国際化 MVP は日本語 UI のみ。文言は将来の i18n を見据えてキー化しておくことが望ましい
NFR-10 保守性 機能仕様と UI 仕様は本ドキュメント群で一元管理。コードからのドキュメント乖離を起こさない

8. ユーザーストーリー

US-01 — フォルダの保険コピーを作りたい

個人 Drive ユーザーとして、
重要な資料が入ったフォルダを大きく編集する前に、
同じ階層に「Copy of …」として完全な複製を作っておきたい。
そうすれば、編集に失敗してもいつでも元の状態に戻せる。

US-02 — 雛形フォルダを新案件用に流用したい

フリーランスとして、
請求書・契約書のテンプレが入った「案件雛形」フォルダを、
新しい案件のたびに複製して名前を付け替えて使いたい。
現状は中身のファイルを 1 個ずつ右クリック→コピーしているのが面倒。

US-03 — 提出前のスナップショットを残したい

学生として、
論文フォルダを教授に提出する直前に 「Copy of 卒論_2026春」 として残しておきたい。
ローカル PC には容量を使いたくない。

US-04 — 大量ファイルでも安心して進めたい

小規模事業者として、
数百ファイル入った業務フォルダを複製するとき、
途中で何件か失敗しても全体は止まらず、最後にどれが失敗したか分かる状態にしたい。

US-05 — 進捗を見ながら待ちたい

すべてのユーザーとして、
複製が走っている間、「今どのファイルを処理中か」「あと何件か」が見えていてほしい。
固まっているのか動いているのかが分からない待ち時間は不安。

9. 受け入れ条件（Acceptance Criteria）

以下を満たした時点で MVP の受け入れ可とする。

AC-01: ログイン

  • 「Google でログイン」ボタンを押すと、Google の認証画面に遷移する
  • 認証後、本アプリのメイン画面に戻り、ヘッダーにログイン中のユーザー名（またはメールアドレス）が表示される
  • 「ログアウト」操作で未認証画面に戻る

AC-02: フォルダ指定とプレビュー

  • 共有 URL 入力欄に有効な Drive フォルダの URL を貼り付けると、自動的にフォルダ名・所在パス・直下のアイテム数のプレビューが表示される（手動「取得」ボタンは存在しない）
  • 無効な URL（フォルダ以外、存在しない、権限なし、共有ドライブ内）の場合、プレビューは表示されず、原因が分かるエラーメッセージが表示される

AC-03: 複製の実行

  • 「複製を開始」を押すと、元フォルダと同じ親階層に「Copy of 〇〇」フォルダが作成される（元フォルダがマイドライブ直下ならマイドライブ直下に作成）
  • 同名フォルダがすでにある場合は Copy of 〇〇 (2)、(3) のように連番が自動付与される
  • 非オーナーの編集可能フォルダの場合、ユーザーが親階層に書き込み権限を持つときのみ作成される。権限がなければ事前に検出して中止し、エラー文言が表示される
  • 元フォルダ内のすべてのサブフォルダ・ファイルが、階層構造を保ったまま複製される
  • Google Docs / Sheets / Slides もそのまま編集可能な状態で複製される
  • 元フォルダ・元ファイルには一切の変更が加えられない（読み取りのみ）

AC-04: 進捗とエラー

  • 処理中、進捗率と現在処理中のアイテム名が画面に表示され続ける。分母（総件数）は処理進行中に増加することがある（再帰列挙と複製を並行するため）
  • 個別ファイルでエラーが発生しても全体は止まらず、次のアイテムに進む
  • 「中止」操作時は進行中の API リクエストを速やかに中断し、途中の複製先フォルダはそのまま残る
  • 完了時（または中止時）にレポート画面が出て、成功件数・スキップ件数・スキップしたアイテム名と理由が一覧できる
  • レポート画面から複製先フォルダを Drive で直接開くリンクがある

AC-05: 安全性

  • 本アプリは元フォルダ・元ファイルに対して書き込み・削除・移動を行わない
  • OAuth で要求するスコープは drive.readonly + drive.file + openid email profile のみ。drive（全権限）スコープは要求しない

本アプリの主要なアクター（ユーザーと外部システム）とユースケースの関係を示す。

ユースケース一覧

11. 制約と前提

  • C-1: Google Drive API のクォータに従う。短期間に大量のリクエストを送ると一時的に 429 エラーが発生する。指数バックオフでのリトライを必須とする。
  • C-2: Drive はフォルダ自体の「コピー」を API でネイティブにはサポートしていない。フォルダは「files.create で空フォルダを再作成」＋「中身を files.copy で複製」の合成で実装する。
  • C-3: 元フォルダの共有設定・コメント・版履歴は MVP では複製しない（files.copy のデフォルト挙動に従う）。
  • C-4: ショートカット（Drive 上のショートカットアイテム）の扱いは MVP では「ショートカットとしてそのまま複製」とし、ショートカット先の実体追従はしない。
  • C-5: 共有ドライブ（チームドライブ）内のフォルダは MVP では対象外。マイドライブ上のフォルダ、および自分が編集権限を持つ他人所有のフォルダ（マイドライブからショートカット等でアクセスできるもの）は対象とする。
  • C-6: 複製先は原則「元フォルダと同じ親階層」に作成する。元フォルダがマイドライブ直下にある場合はマイドライブ直下に作成。非オーナーフォルダで親階層に書き込み権限がない場合は、自動フォールバックを行わずエラーとして中止する（ユーザーが意図しない場所に複製が生まれるのを防ぐため）。
  • C-7: 巨大フォルダ（数千ファイル以上）の複製は MVP のテスト対象外。技術的に動くとしても UX 上の保証はしない。

12. 非スコープ（やらないこと）

  • ローカル PC への書き出し・ダウンロード（本アプリは Drive 内で完結する）
  • 双方向同期・差分同期・自動定期実行
  • 共有ドライブ・他人のドライブへの操作
  • ファイルの内容変換（PDF 化等）
  • 共有権限・コメント・版履歴の複製
  • モバイルブラウザでの最適化（PC ブラウザを優先）
  • オフラインでの実行
