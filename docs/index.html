プロジェクト概要

Google Drive バックアップ は、Drive 内のフォルダをまるごと「Copy of 元フォルダ名」として同じ親階層に複製する Web アプリ。 ローカル PC のストレージを使わず、Drive 内で完結するスナップショット作成・雛形流用を可能にする。

本プロジェクトの根本方針は CLAUDE.md に従い 「作るものをはっきりさせてから AI のコーディングを始める」。 実装に着手する前に、以下のドキュメント群と作業単位ステアリングがすべて承認済みであることを確認する。

作業単位のドキュメント（.steering/）

特定の作業について「今回何をするか」を定義する一時的なドキュメント。 ディレクトリ命名: .steering/YYYYMMDD-開発タイトル/。 各ディレクトリには次の 3 ファイルが入る：

  • requirements.html — 今回の要求内容・ユーザーストーリー・受け入れ条件・制約
  • design.html — 実装アプローチ・変更コンポーネント・データ構造変更・影響範囲。UI 影響セクション必須
  • tasklist.md — 実装タスク・進捗・完了条件

本リポジトリ初回セットアップ後の作業単位（予定）:

  • .steering/20260528-initial-implementation/ — MVP 初回実装（このあと作成）

共通アセット

  • assets/design-system.css — 全 HTML が参照する共通スタイル（デザイントークン定義含む）
  • assets/template.html — 新規 HTML 作成時の雛形
  • assets/images/ — 画像（PNG / SVG）配置先（現在空）
