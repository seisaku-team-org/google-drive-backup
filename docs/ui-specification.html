0. デザイン原則 / トーン

  1. 「迷わない 1 本道」 — MVP は「ログイン→URL貼付→確認→実行→完了」の単線フロー。分岐は最小限にし、各画面で次に押すべきボタンを 1 つに絞る。
  2. 「壊さない安心感」 — このツールは元データに書き込まない。画面内に「元フォルダには変更を加えません」と明示する文言を 1 箇所以上配置する。
  3. 「待ち時間を可視化」 — Drive API は遅い処理がある。空白の待ちを作らない（スピナー＋現在処理中ファイル名＋件数を常時表示）。
  4. 「失敗をスキップして前に進む」 — 個別エラーで止めない。最後にレポートでまとめて見せる。
  5. 「PC 中心、モバイルでも動く」 — メイン UX は PC ブラウザ。モバイルでもレイアウト崩れなく動作するが、専用最適化はしない。
  6. 「日本語、です/ます調」 — 用語は 用語集 に従い、UI ラベルと用語集の対応を 7. 用語整合 で管理する。
  7. 「アクセシビリティは最低 WCAG 2.1 AA」 — キーボードのみで全操作可能、コントラスト 4.5:1 以上、フォーカス可視。

2. ユーザーフロー / 画面遷移図

2.1 主フロー（ハッピーパス）

  1. SCR-001-login で「Google でログイン」を押す → OAuth → 認証成功で SCR-002-home へ
  2. SCR-002-home で複製したいフォルダの共有 URL を貼り付け → 「フォルダ情報を取得」 → プレビュー表示
  3. プレビュー内容を確認し「複製を開始」 → SCR-003-progress へ
  4. 進捗を見ながら完了を待つ → 自動的に SCR-004-report へ
  5. レポートで複製先フォルダを Drive で開くか、「新しい複製を始める」で SCR-002-home に戻る

2.2 補助フロー

  • ログアウト: 任意の画面のヘッダーから実行可能。SCR-001-login に戻る（複製処理中は確認ダイアログを出す）
  • トークン失効: API 呼び出しで 401 を受けたら自動的に SCR-001-login に戻り、トーストで通知
  • 中止: SCR-003-progress で「中止」を押すと、現在処理中のアイテムを完了してから停止し SCR-004-report へ

3.2.6 アクセシビリティ

  • URL 入力欄は <input type="url">、ラベルは <label for> で紐付け

3.3 SCR-003-progress: 複製中（進捗）

3.3.1 目的

複製処理の進捗をリアルタイムに表示し、ユーザーに「動いている」安心感と「あとどれくらいで終わるか」の見通しを与える。任意のタイミングで中止できる。

進捗の 2 フェーズ表示（2026-05-29 設計確定）

動作確認の結果「分母が処理中に増える」挙動はユーザーが終わりを予測できず使いづらいと判明したため、進捗を 2 フェーズに分けて表示する：

  1. 集計フェーズ（counting）: 元フォルダを再帰的に列挙して総アイテム数を確定する。UI は「集計中…」見出し、スピナー、「N 件 発見」カウンター、列挙中のフォルダ名を表示。進捗バーは出さない（total が未確定のため）
  2. 複製フェーズ（copying）: 確定した total に対して進捗バーを 0%→100% に正確に進める。「45% (12 / 27)」のように決定的な数値を表示

列挙キャッシュを内部で持つため、API 呼び出し回数は 1 フェーズ方式と同じ（1 フォルダにつき listAllChildren を 1 回のみ）。複製先フォルダの作成は集計後に行うため、集計中に中止した場合 Drive にゴミは残らない。

3.3.6 アクセシビリティ

4. 共通コンポーネント

全画面で再利用するコンポーネントを定義する。色味・サイズの実装値は 5. デザイントークン 参照。

4.1 ボタン

種別用途例
Primaryその画面で最も推奨される操作（CTA）「複製を開始」「Google でログイン」
Secondary準次操作「情報を取得」
Tertiary (link)補助操作「キャンセル」「ログアウト」「レポートをコピー」
Danger破壊的・引き返しにくい操作「中止」

状態: 通常 / hover / focus / active / disabled / busy（loading 中、内部にスピナーを置く）。

4.2 フォーム要素

  • 入力欄は <label> 必須。プレースホルダだけのラベル代用は不可（アクセシビリティ要件）。
  • エラー時は入力欄の枠を赤に、下に赤字でエラー文を出す（aria-describedby で関連付け）。
  • 入力欄は読み込み中・処理中は disabled。

4.3 モーダル（確認ダイアログ）

用途: 「中止」確認、ログアウト時の確認（処理中の場合）。構造: 見出し ＋ 説明文 ＋ 2 ボタン（Cancel / Confirm）。Esc キーで閉じる、外側クリックでも閉じる（破壊的アクションの確認時は外側クリックは無効化）。

4.4 トースト

用途: 「ログインがキャンセルされました」「クリップボードにコピーしました」「セッションが切れました（再ログインしてください）」など、ユーザーに非ブロッキングで通知。位置: 画面右下（モバイルは下端中央）。表示時間: 4 秒（重要度高は 6 秒）。手動で閉じる × ボタン。

4.5 進捗バー

用途: SCR-003-progress 専用。<progress max="100" value="..."> または role="progressbar" 実装。

4.6 カード

用途: プレビューカード、カウンターカード、スキップ詳細カード。背景 --color-surface、角丸 6px、内側 padding 0.75rem 1rem。

4.7 リンク

本文中のリンク・複製先リンクは --color-accent、hover で下線。新しいタブで開くリンクには ↗ アイコンを末尾に付与。
